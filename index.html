03. 通信基础

Posted on 2020-12-15 In 计算机科学与技术 , 计算机网络 (考研)

物理层接口特性

  1. 解决如何在连接各种计算机的传输媒体传输上传输比特流，而不是指具体的传输模块
  2. 主要任务是确定与传输媒体接口有关的一些特性，定义标准
  3. 机械特性：定义物理连接的特性，规定物理连接时所采用的规格、接口形状、引线数目、引脚数量和排列情况
  4. 电气特性：规定传输二进制位时，线路上信号的电压范围、阻抗匹配、传输速率和距离限制
  5. 功能特性：指明某条线上出现的某一电平表示何种意义，接口部件的信号线的用途
  6. 规程特性 (过程特性)：定义各条物理线路的工作规程和时序关系

数据通信模型

  1. 输入信息经过信源变为输入数据
    输入数据经过发送器变为发送信号
    发送信号经过传输系统变为接受信号
    接受信号经过接收器变为输出数据
    输出数据经过信宿变为输出信息
  2. 从开始到发送器成为源系统；从接收器到最后称为目的系统；中间的是传输系统
  3. 通信的目的是传送消息
    数据：传送信息的实体，通常是有意义的符号序列
    数据通信：在不同计算机之间传输表示信息的二进制数0、1序列的过程
  4. 信号：数据的电气、电磁的表现，是数据在传输过程中的存在形式
    数字信号 / 离散信号：代表消息的参数的取值是离散的
    模拟信号 / 连续信号：代表消息的参数的取值是连续的
  5. 信源：产生恶化发送数据的源头
    信宿：接受数据的终点
  6. 信道：信号的传输媒介。一般用来表示向某一个方向传送信息的介质，因此一条通信线路往往包含一条发送信道和一条接受信道。信道是单向的
    按传输信号分：模拟信道、数字信道
    按传输介质分：无线信道、有线信道
  7. 单工通信：只有一个方向的通信而没有反方向的交互，仅需要一条信道
    半双工通信 / 双向交替通信：通信的双方都可以发送货接受信息，但任何一方都不能同时发送和接受，需要两条信道
    全双工通信 / 双向同时通信：通信双方可以同时发送和接受信息，需要两条信道
  8. 串行传输：将表示一个字符的8位二进制数按低位到高位的顺序依次发送。速度慢，费用低，适合远距离
    并行传输：将表达一个字符的8位二进制数同时通过8条信道发送。速度快，费用高，适合近距离
  9. 同步传输 (区块传输)：数据的传送是以一个数据区块为单位，在传送数据时，需先送出1个或多个同步字符，再送出整批的数据。传送数据时，加多个同步字符
    异步传输：将比特分为小组进行传送，小组可以是8位或更长。发送方可以在任何时刻发送这些比特组，而接收方不知道它们会在什么时候到达。传送数据时，加一个字符起始位和一个字符终止位
  10. 码元：用一个固定时长的信号波形 (数字脉冲)，代表不同离散数值的基本波形，是数字通信中数字信号的计量单位，这个时长内的信号称为$k$进制码元，该时长称为码元宽度。当码元的离散状态有$M$个时，码元为$M$进制码元
    码元是基本波形，波形每一次变动都是一个码元
    一个码元可以携带多个bit的信息量，$M$进制码元可以携带$log_2M$个bit的信息量
  11. 速率 / 数据率：数据的传输速率，表示单位时间内传输的数据量
    波特率 / 码元传输速率 / 码元速率 / 波形速率 / 调制速率 / 符号速率：单位时间内数字通信系统所传输的码元个数 (脉冲个数 / 信号变化个数)。单位是波特 ($Baud / B$)
    波特率与进制数无关，只与码元宽度$T$有关
    比特率 (信息传输速率 / 信息速率)：单位时间内数字通信系统传输的比特数。单位是$b/s$
    若一个码元携带的信息量，则的波特率所对应的比特率为
  12. 带宽：单位时间从网络中的一点到另一点所能通过的最高数据率，或单位时间内通过链路的数量。单位是$bps$

02. 体系结构和参考模型

Posted on 2020-12-06 In 计算机科学与技术 , 计算机网络 (考研)

体系结构

  1. 发送文件前要完成的工作：

    发起通信的计算机必须将数据通信的通路进行激活

    要告诉网络如何识别目标主机

    查明目的主机是否开机，并且网络连接正常

    弄清楚对方计算机中文件管理程序是否已经做好准备

    确保差错和意外可以解决

  2. 分层的体系结构把大问题化为多个小问题逐一解决

  3. 实体：每一层中的活动对象

    对等实体：同一层的实体

  4. 协议：对等实体之间的一系列规定

    协议三部分：语法规定传输数据的格式，语义规定所要完成的功能，同步规定各种操作的顺序

  5. 接口：连接相邻两层之间的上层使用下层服务的逻辑入口

    服务：下层为相邻上层提供的功能调用

  6. 分层的要求

    各层之间相互独立，每层只实现一种相对独立的功能

    每层之间界面自然清晰，易于理解，相互交流尽可能少

    结构上可分割开，每层都采用最合适的技术实现

    保持下层对上层的独立性，上层单向使用下层的服务

    分层有助于标准化工作的进行

  7. SDU服务数据单元：为完成用户所要求的功能而应传送的数据

    PCI协议控制信息：控制协议操作的信息

    PDU协议数据单元：对等层次之间传送的数据单元

参考模型

OSI参考模型

  1. OSI参考模型是法定标准

  2. OSI模型由七层组成，自上而下依次是：应用层、表示层、会话层、传输层、网络层、数据链路层、物理层

  3. 通信的两个端系统都要经历完整的七层，中间设备只要两层或三层

    最上面四层无需中间设备参与，是端到端通信；下三层是点到点通信

  4. 数据的处理

    物理层不对数据做处理，只转换成信号并传输

    发送端除物理层外的每层会把上层的传下来的数据添加一个首部，包装成该层的PDU。数字链路层除了首部还添加了一个尾部

    接收端每层都把下一层传来的数据处理解码，去除发送端对应层添加的内容，再发送给上一层

  5. 应用层：所有能和用户交互产生网络流量的程序，即需要联网的应用程序

    典型应用层服务：文件传输 (FTP协议)、电子邮件 (SMTP协议)、万维网 (HTTP协议)

  6. 表示层：处理在两个通信系统中交换信息的表示方式 (语法和语义)

    功能：数据格式变换、数据加密解密、数据压缩和恢复

  7. 会话层：向表示层实体/用户进程提供建立连接并在连接上有序地传输数据

    会话 / 建立同步 (SYN)：建立连接并有序传输

    功能：建立、管理、终止会话、使用校验点可使会话在通信失效时从校验点 / 同步点继续恢复通信，实现数据同步

    主要协议：ADSP、ASP

  8. 传输层：负载主机中两个进程的通信，即端到端的通信。端表示的是端口号

    传输单位是报文段或用户数据报

    功能：可靠传输、不可靠传输、差错控制、流量控制、复用分用

    复用：多个应用层进程同时使用下面运输层的服务

    分用：运输层把收到的信息依据端口号分别交给上面应用层中的相应进程

    主要协议：TCP、UDP

  9. 网络层：把分组从原端传到目的端，为分组交换网上的不同主机提供通信服务

    传输单位是数据报

    功能：路由选择 (最佳路径)、流量控制、差错控制、拥塞控制

    拥塞状态：所有节点都来不及接受分组，而要丢弃大量分组

    主要协议：IP、IPX、ICMP、IGMP、ARP、RARP、OSPF

  10. 数据链路层：把网络层传下来的数据报组装成帧

    传输单位是帧

    功能：成帧 (定义帧的开始和结束)、差错控制 (帧错、位错)、流量控制、访问 (对信道的访问接入)控制

    主要协议：SDLC、HDLC、PPP、STP

  11. 物理层：在物理媒体上实现比特流的透明传输

    传输单位是bit

    透明传输：不管所传数据是什么样的bit组合，都应当能够在链路上传送

    功能：定义接口特性、定义传输模式 (单工、半双工、双工)、定义传输速率、bit同步、bit编码

    主要协议：Rj45、802.3

TCP/IP模型

  1. TCP/IP模型是事实标准

  2. TCP/IP模型有四层，自上而下分别是应用层、传输层、网际层、网络接口层

  3. TCP/IP模型的应用层对应OSI模型的应用层、表示层、会话层，传输层对应传输层，网际层对应网络层，网络接口层对应数据链路层、物理层

  4. 面向连接三个阶段：$建立连接\rightarrow数据传输\rightarrow释放连接$

    无连接直接数据传输

OSI模型 TCP/IP模型
网络层 无连接+面向连接 无连接
传输层 面向连接 无连接+面向连接
  1. 5层参考模型：自上而下分别是应用层、传输层、网络层、数据链路层、物理层

01. 计算机网络概述

Posted on 2020-12-05 Edited on 2020-12-06 In 计算机科学与技术 , 计算机网络 (考研)

概念与功能

概念

  1. 计算机系统：手机、电脑等终端
  2. 通信设备：路由器、集线器、中继器、交换机
  3. 线路：逻辑线路，不一定要是实体线路，可以利用无线方式实现
  4. 计算机网络：将分散的、具有独立功能的计算机系统，通过通信设备与线路连接起来，由功能完善的软件实现资源共享和信息传递的系统
  5. 计算机网络是互连的、自治的计算机集合，通过通信链路互联互通，无主从关系

功能

  1. 数据通信
  2. 资源共享： 资源包括硬件、软件、数据，同一个网络上的其他计算机都可以使用
  3. 分布式处理：多台计算机各自承担同一工作任务的不同部分
  4. 提高可靠性
  5. 负载均衡

发展历史

  1. 第一阶段

    美苏冷战时期，美国国防部高级研究计划局 (ARPA)设计的一个分散的指挥系统，阿帕网 (ARPAnet)

    阿帕网希望实现不同网络互连 (internet)，接受TCP/IP协议，形成Internet

    网络连接计算机，互联网连接网络，Internet是最大的互联网

  2. 第二阶段

    美国国家科学基金会建立的三层网络

    最低层为校园网，校园网需要连接地区网，地区网连接主干网

  3. 第三阶段

    ISP：Internet服务提供者，Internet服务提供商

    本质还是三层网络，但是ISP从美国政府分散到各个公司

    访问流程：$主机 \rightarrow 本地ISP\rightarrow 地区ISP\rightarrow\dots\rightarrow 主干ISP\rightarrow\dots\rightarrow 地区ISP\rightarrow 本地ISP\rightarrow 主机$

    IXP：Internet交换点，让两个网络可以直接相连，无需向上传递数据可以直接通信

组成

  1. 组成部分：硬件、软件、协议 (一系列规定和约定的集合)

  2. 硬件组成了网络的端系统以及中间设备

    软件附着在端系统上，部分在中间设备上

    协议规定数据如何封装打包，如何在网络中传播

  3. 边缘部分：端系统，用户直接使用

    核心部分：为边缘部分服务的中间设备

  4. 边缘部分端系统之间的通信是进程之间的通信

  5. 边缘部分之间的通信方式

    C/S方式：Client to Server，把数据从发起Client都发送到Server上再发到目标Client，Client越多，速度越慢

    B/S方式：Browser to Server，与C/S方式类似，但是发生在Browser上

    P2P方式：Peer to Peer，所有主机都是提供服务的Server，也是享受服务的Client，使用的主机越多，速度越快

  6. 通信子网：实现数据通信，由各种传输介质、通信设备、相应的网络协议组成

    资源子网：实现资源共享 / 数据处理，由实现资源共享功能的设备和软件的集合组成

    在OSI七层模型中，下三层 (物理层、数据链路层、网络层)是通信子网，上三层 (会话层、表示层、应用层)是资源子网，传输层连接资源子网和通信子网

分类

  1. 按分布范围分：广域网 (WAN)、城域网 (MAN)、局域网 (LAN)、个人区域网 (PAN)

    广域网使用交换技术，局域网使用广播技术

    不看地理范围，而看使用的技术，即使只有几米，使用广域网技术，也是广域网

  2. 按使用者分：公用网、专用网

  3. 按交换技术分：电路交换、报文交换、分组交换

    电路交换：$建立连接，占用资源\rightarrow 通信，持续占用资源\rightarrow 通信结束，释放资源$

    电路交换在通信时别人无法与通信中的人通信

    报文交换、分组交换：使用存储转发技术，不会占线，区别是发送的报文是否分组

  4. 按拓扑结构分：总线型、星型、环型、网络型

    星型结构中如果有$n$个节点，则有$n-1$条链路

  5. 按传输技术分：广播式网络、点对点网络

    广播式网络是共享公共通信信道，网络中所有人都可以收到信息

    点对点网络是使用分组存储转发和路由选择机制，只有目标才可以收到信息

标准化工作

  1. 标准分类

    法定标准：由权威机构制定的正式的、合法的标准

    事实标准：某些公司的产品在竞争中占据了主流，时间长了，这些产品中的协议和技术就成了标准

  2. 标准的制定

    RFC (Request For Comment)：Internet标准的形式。所有的Internet标准都是RFC格式，但不是所有RFC都是Internet标准

    RFC成为Internet正式标准：

    Internet草案还不是RFC文档，从建议标准开始才是

    草案标准已经取消

性能指标

速率、带宽、吞吐量

  1. 速率 / 数据率 / 数据传输率 / 比特率：连接在计算机网络上的主机在数字信道上传送数据位数的速率

  2. 单位：b/s、kb/s、Mb/s、Gb/s、Tb/s

    $1Tb/s=10^3Gb/s=10^6Mb/s=10^9kb/s=10^{12}b/s$

    注意与内存的转化区别开，内存中每级差$2^{10}=1024$

  3. 带宽：单位时间从网络中的一点到另一点所能通过的最高数据率

  4. 吞吐量：单位时间内通过某个网络的数据量

  5. 吞吐量受网络的带宽限制，带宽是吞吐量的上限

时延

  1. 时延 / 延迟 / 迟延：数据从网络的一端传送到另一端所需的时间

  2. 时延由四部分组成

    发送时延：发送分组的第一个bit到分组的最后一个bit发送完所需的时间，发生在发送端内。$发送时延=\displaystyle\frac{数据长度}{信道带宽}$

    传播时延：在链路上传播花费的时间，发生在链路上。$传播时延=\displaystyle\frac{信道长度}{电磁波在信道上的传播速率}$

    排队时延：等待输出/输入链路可用的时间，发生在中间设备中

    处理时延：在中间设备中检错和找出口，发生在中间设备中

  3. 高速链路提高了信道带宽，缩短了发送时延

  4. $时延带宽积=传播时延\times带宽$，单位是bit，表示某段链路上现在有多少bit。又称为以bit为单位的链路长度

  5. 往返时延RTT：从发送方发送数据开始，到发送方收到接收方的确认 (收到数据后立即发送确认)总共经历的时延

    RTT越大，在收到确认之前，可以发送的数据越多

    $RTT=传播时延\times 2+末端处理时间$

  6. $信道利用率=\displaystyle\frac{有数据通过的时间}{总时间}$

    网络利用率：信道利用率的加权平均值

  7. 时延随利用率的提升而提高，且在利用率接近$1$时急剧上升
