01. 计算机网络概述

Posted on 2020-12-05 Edited on 2021-01-27 In 考研 , 计算机网络

概念与功能

概念

  1. 计算机系统：手机、电脑等终端
  2. 通信设备：路由器、集线器、中继器、交换机
  3. 线路：逻辑线路，不一定要是实体线路，可以利用无线方式实现
  4. 计算机网络：将分散的、具有独立功能的计算机系统，通过通信设备与线路连接起来，由功能完善的软件实现资源共享和信息传递的系统
  5. 计算机网络是互连的、自治的计算机集合，通过通信链路互联互通，无主从关系

功能

  1. 数据通信
  2. 资源共享： 资源包括硬件、软件、数据，同一个网络上的其他计算机都可以使用
  3. 分布式处理：多台计算机各自承担同一工作任务的不同部分
  4. 提高可靠性
  5. 负载均衡

发展历史

  1. 第一阶段
    美苏冷战时期，美国国防部高级研究计划局 (ARPA)设计的一个分散的指挥系统，阿帕网 (ARPAnet)
    阿帕网希望实现不同网络互连 (internet)，接受TCP/IP协议，形成Internet
    网络连接计算机，互联网连接网络，Internet是最大的互联网
  2. 第二阶段
    美国国家科学基金会建立的三层网络
    最低层为校园网，校园网需要连接地区网，地区网连接主干网
  3. 第三阶段
    ISP：Internet服务提供者，Internet服务提供商
    本质还是三层网络，但是ISP从美国政府分散到各个公司
    访问流程：$主机 \rightarrow 本地ISP\rightarrow 地区ISP\rightarrow\dots\rightarrow 主干ISP\rightarrow\dots\rightarrow 地区ISP\rightarrow 本地ISP\rightarrow 主机$
    IXP：Internet交换点，让两个网络可以直接相连，无需向上传递数据可以直接通信

组成

  1. 组成部分：硬件、软件、协议 (一系列规定和约定的集合)
  2. 硬件组成了网络的端系统以及中间设备
    软件附着在端系统上，部分在中间设备上
    协议规定数据如何封装打包，如何在网络中传播
  3. 边缘部分：端系统，用户直接使用
    核心部分：为边缘部分服务的中间设备
  4. 边缘部分端系统之间的通信是进程之间的通信
  5. 边缘部分之间的通信方式
    C/S方式：Client to Server，把数据从发起Client都发送到Server上再发到目标Client，Client越多，速度越慢
    B/S方式：Browser to Server，与C/S方式类似，但是发生在Browser上
    P2P方式：Peer to Peer，所有主机都是提供服务的Server，也是享受服务的Client，使用的主机越多，速度越快
  6. 通信子网：实现数据通信，由各种传输介质、通信设备、相应的网络协议组成
    资源子网：实现资源共享 / 数据处理，由实现资源共享功能的设备和软件的集合组成
    在OSI七层模型中，下三层 (物理层、数据链路层、网络层)是通信子网，上三层 (会话层、表示层、应用层)是资源子网，传输层连接资源子网和通信子网

分类

  1. 按分布范围分：广域网 (WAN)、城域网 (MAN)、局域网 (LAN)、个人区域网 (PAN)
    广域网使用交换技术，局域网使用广播技术
    不看地理范围，而看使用的技术，即使只有几米，使用广域网技术，也是广域网
  2. 按使用者分：公用网、专用网
  3. 按交换技术分：电路交换、报文交换、分组交换
    电路交换：$建立连接，占用资源\rightarrow 通信，持续占用资源\rightarrow 通信结束，释放资源$
    电路交换在通信时别人无法与通信中的人通信
    报文交换、分组交换：使用存储转发技术，不会占线，区别是发送的报文是否分组
  4. 按拓扑结构分：总线型、星型、环型、网络型
    星型结构中如果有$n$个节点，则有$n-1$条链路
  5. 按传输技术分：广播式网络、点对点网络
    广播式网络是共享公共通信信道，网络中所有人都可以收到信息
    点对点网络是使用分组存储转发和路由选择机制，只有目标才可以收到信息

标准化工作

  1. 标准分类
    法定标准：由权威机构制定的正式的、合法的标准
    事实标准：某些公司的产品在竞争中占据了主流，时间长了，这些产品中的协议和技术就成了标准
  2. 标准的制定
    RFC (Request For Comment)：Internet标准的形式。所有的Internet标准都是RFC格式，但不是所有RFC都是Internet标准
    RFC成为Internet正式标准：
    Internet草案还不是RFC文档，从建议标准开始才是
    草案标准已经取消

性能指标

速率、带宽、吞吐量

  1. 速率 / 数据率 / 数据传输率 / 比特率：连接在计算机网络上的主机在数字信道上传送数据位数的速率
  2. 单位：b/s、kb/s、Mb/s、Gb/s、Tb/s
    $1Tb/s=10^3Gb/s=10^6Mb/s=10^9kb/s=10^{12}b/s$
    注意与内存的转化区别开，内存中每级差$2^{10}=1024$
  3. 带宽：单位时间从网络中的一点到另一点所能通过的最高数据率
  4. 吞吐量：单位时间内通过某个网络的数据量
  5. 吞吐量受网络的带宽限制，带宽是吞吐量的上限

时延

  1. 时延 / 延迟 / 迟延：数据从网络的一端传送到另一端所需的时间
  2. 时延由四部分组成
    发送时延：发送分组的第一个bit到分组的最后一个bit发送完所需的时间，发生在发送端内。$发送时延=\displaystyle\frac{数据长度}{信道带宽}$
    传播时延：在链路上传播花费的时间，发生在链路上。$传播时延=\displaystyle\frac{信道长度}{电磁波在信道上的传播速率}$
    排队时延：等待输出/输入链路可用的时间，发生在中间设备中
    处理时延：在中间设备中检错和找出口，发生在中间设备中
  3. 高速链路提高了信道带宽，缩短了发送时延
  4. $时延带宽积=传播时延\times带宽$，单位是bit，表示某段链路上现在有多少bit。又称为以bit为单位的链路长度
  5. 往返时延RTT：从发送方发送数据开始，到发送方收到接收方的确认 (收到数据后立即发送确认)总共经历的时延
    RTT越大，在收到确认之前，可以发送的数据越多
    $RTT=传播时延\times 2+末端处理时间$
  6. $信道利用率=\displaystyle\frac{有数据通过的时间}{总时间}$
    网络利用率：信道利用率的加权平均值
  7. 时延随利用率的提升而提高，且在利用率接近$1$时急剧上升
