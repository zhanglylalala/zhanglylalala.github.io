动态和静态

操作

  1. 动态操作：需写入，改变元素个数，如：insert、remove
  2. 静态操作：仅读取，不改变元素个数

存储方式

  1. 静态存储：物理存储次序与其逻辑次序严格一致，比如向量、数组
  2. 动态存储：为各元素动态分配和回收物理空间，逻辑上相邻的元素记录彼此的物理地址，逻辑上形成整体，比如列表
  3. 动态存储可支持高效的动态操作，但静态操作极其低效；静态存储可支持高效的静态操作，但动态操作极其低效

列表

元素称为节点，各节点利用指针或引用彼此连接，在逻辑上构成一个线性序列
相邻节点彼此互称前驱、后继，前驱或后继若存在，必唯一。没有前驱或后继的唯一节点称作首/末节点
列表各个元素物理位置不在与逻辑位置相关，所以用循位置访问，利用节点间的互相引用访问元素
ADT接口

为了方便实现，在首节点之前插入一个头节点，在末节点之后插入一个尾节点作为哨兵节点
构造方式：新建一个头节点和一个尾节点，把他们相互连接起来

静态操作

访问元素：重载[r]，从首节点(注意，不是头节点)开始，顺数第r个元素(取r次后继)
查找：某一个元素p的n个真前驱中查找某个可能存在的元素e。从p开始，不断取前驱，与e比较。相等返回，没找到返回NULL

动态操作

插入

  1. 以插入在一个元素的前面为例，插在后面类似
  2. 步骤：新建节点 -> 设置新节点的前驱后继 -> 更新两个旧节点的前驱后继
  3. 得益于哨兵节点的存在，插入操作永远合法，首节点也一定有前驱，末节点也一定有后继
  4. 基于插入的复制构造：依据另一个列表，不断插入

删除

  1. 步骤：备份节点 -> 将p的前驱作为p的后继的前驱 -> 将p的后继作为p的前驱的后继 -> delete p
  2. 析构：不断删除可见节点，再释放头尾节点
  3. 唯一化：和向量类似，向前找是否存在和当前元素一样的节点，但是要注意，删除的是前面的节点而不是当前正在访问的节点，因为删除时要释放节点，如果删除当前节点会使当前指针变为野指针

动态操作的复杂度几乎是O(1)，但是其中涉及到的new和delete所需要的时间大概是一般操作(如加法)的100倍
