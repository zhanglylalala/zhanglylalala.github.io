Tricks in Xcode

Posted on 2019-07-17 Edited on 2020-04-04 In Computer Science , Developing iOS App with Swift (Stanford CS193p)

Debugger

During a Debugger, in the Console, after the word (lldg), we can use some syntax to print something we want.
po + sth. print an object
p + sth. print like in swift

Key command in Xcode

Function Key Command
Run project cmd + R
show the left pane (Nevigator) cmd + 0
show the right pane cmd + option + 0
In Navigator, change through the tabs cmd + 1, 2, 3…
In right pane, change through the tabs cmd + option + 1, 2, 3…
show the Debugger and Console cmd + shift + y
show the associate editor cmd + option + enter
open file cmd + shigt + O then search the file (It's better to choose Uses focused editor in preference -> Navigation -> Navigation)
mark position A comment start with "MARK: ", It will than shows in the navigation above, where it shows where we are now
move forward a tab cmd + [
move backward a tab cmd + ]
make every lines have the right number of tabs (like python) select and cmd + i

Something Else

  1. If we draw something in code, it won’t show in storyboard by default. But we make it show by adding @IBDesignable right before the declare of the drawing view class.
  2. We can even make some properties inspectable in the attributed inspector by adding @IBInspectable right before the declare in the class.
  3. When some objects overlap at Interface Builder that we can hardly create outlet for them, we can either use the Document Outline or shift-click to choose the object.
