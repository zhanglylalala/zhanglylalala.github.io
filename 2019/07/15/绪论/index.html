什么是算法

算法：特定计算模型下，旨在解决特定问题的指令序列

算法的特点

  1. 输入、输出
  2. 正确性
  3. 确定性
  4. 可行性
  5. 有穷性

评价算法

评价内容

  1. 算法分析的两个主要方面：正确性和成本 (运行时间 + 所需存储空间)
  2. 评价度量：TA(n) = 算法A在求解某一问题所有规模为n的实例时所需要的最大成本

图灵机模型

  1. 要素：纸( 默认全为#，无限空间 )、表头、指令
  2. 图灵机就是表头在纸带上各个单元格中根据当前格的状态执行不同的指令
  3. 指令格式：
    读写头当前状态(上一次移动的方向< / >q
    读写头正对单元格的状态c
    当前单元格修改后的状态d
    读写头的移动方向L/R
    读写头下一个状态(同q)p)
  4. 当图灵机达到特定的状态h时就停机
  5. 例子：二进制加一
     (<, 1, 0, L, <) 左行，1 -> 0
     (<, 0, 1, R, >) 掉头，0 -> 1
     (<, #, 1, R, >) 超出当前长度
     (>, 0, 0, R, >) 右行
     (>, #, #, L, h) 复位
  6. RAM模型：无限寄存器，每个寄存器内存储不同的内容，每次操作都是对不同的寄存器做一次常数时间的操作
  7. 算法所需的时间正比于执行的基本操作次数，所以取成本的计算方式为利用图灵机模型或RAM模型完成算法的基本操作的次数

渐进分析

概述

  1. 渐进分析：问题规模足够大之后，计算成本如何增长，即n >> 2后，对于规模为n的输入的T(n)
  2. 渐进分析在n不够大，在局部细微的反面是不够精确的，但在n足够大的情况下，在宏观来看渐进分析是准确的。
  3. 渐进分析是对增长趋势的大致分析，不是对操作次数的精确分析

大O记号

  1. 定义：如果∃c，当n >> 2后，有T(n) < c · f(n)，则记T(n) = O( f(n) )
  2. 大O记号的运算规则：O( c · f(n) ) = O( f(n) ) O( na + nb ) = O( nb ) (当0 < a < b)
  3. 大O记号只有在n足够大的情况下才可以构成T(n)的上界，在小范围内T(n)可能大于f(n)
  4. 大Ω记号：如果∃c，当n >> 2后，有T(n) > c · f(n)，则记T(n) = Ω( f(n) )
  5. 大𝚯记号：如果∃c1 > c2 > 0，当n >> 2后，有c1 · f(n) > T(n) > c2 · f(n)，则记T(n) = 𝚯( f(n) )

算法分析

  1. 两个主要任务：正确性(不变性、单调性)与复杂度
  2. 空间复杂度一般不考虑输入空间
  3. 就渐进复杂度的意义而言，在任一算法的任何一次运行过程中所消耗的存储空间，都不会多于其间所执行基本操作的累计次数
  4. 每次基本操作所涉及的存储空间都不会超过常数规模
  5. 时间复杂度本身就是空间复杂度的一个天然的上界

复杂度分析

  1. 迭代：级数求和
  2. 递归：递归跟踪 + 递归方程

级数

  1. 算术级数：与末项平方同阶 e.g. T(n) = 1 + 2 + … + n = O(n)
  2. 幂方级数：比幂次高一阶 e.g. T(n) = 12 + 22 + … + n2 = O(n3)
  3. 几何级数：与末项同阶 e.g. T(n) = a0 + a1 + … + an = O(an)
  4. 收敛级数：O(1) e.g. T(n) = 1/1/2 + 1/2/3 + … + 1/(n - 1) / n = 1 - 1 / n = O(1)
  5. 调和级数：𝚯(log n) e.g. T(n) = 1 + 1/2 + … + 1/n = 𝚯(log n)
  6. 对数级数：𝚯(nlog n) e.g. T(n) = log 1 + log 2 + … + log n = log(n!) = 𝚯(nlog n)

    递归

  7. 递归树
  8. 递归方程 + 递归基方程
    e.g.T(n) = T(n - 1) + O(1) 递归方程
      T(0) = O(1) 递归基方程
