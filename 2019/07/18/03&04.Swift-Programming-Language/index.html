Teaser of Autolayout

How to set constraints of a button or label in the View?

Hold on Ctrl and drag from the button or label to the View, and select the kind of constraints we need.

How to keep a group of members change their sizes to adopt the change of screen size?

  1. Embed them into a stack view. Mark thar a stack view cannot fix in both axises, so we usually creat stack in one direction and create a stack of those stacks in a different axises.
  2. After that, we can set some datas of the stacks. The window just like below. The first selection is the way to keep members in order, to lie them all horizantally or vertically.
    The second selection is the way the whole stack view shows, fill the whole allowed space or in a peticular space.
    The third one is the way to distribute the size of the members.
    The forth one is the space between each members

    How to choose what kind of constraint we want?
  3. Pin to the top: Top space to safe area
    Pin to the buttom: Buttom space to safe area
    Pin to the left: Leading space to safe area
    Pin to the right: Trailing space to safe area
  4. If want to keep in the center, choose “Center xxx in safe area”
  5. If want to keep a certain distance between to members, drag to the other member and choose the vertical spacing or horizantal spacing
  6. We can even change the parameter of the constraints by click the blue line represents the constraint. There are two important parameters, the Relation(second one) and the Constant(the forth one). They mark the constraint between the First Item and the Second Item

Swift

When will we use the Computed Properties?

If one property is derived(calculated) from other state, then we will make it Computed Property. We do that to make sure those properties are synced, to prevent we forget to sync them. For example, the Int? we used in the Consentrationg to record if there is one and only card that is faced up. Its value is derived from the Card array.

Access Control

  1. internal: this is the default, we don’t need to write it down. It means “usable by any object in my app or framework” private - this means “only callable from within this object”
  2. private(set): this means “this property is readable outside this object, but not settable” fileprivate - accessible by any code in this source file
  3. public: (for frameworks only) this can be used by objects outside my framework
  4. open: (for frameworks only) public and objects outside my framework can subclass this
  5. Mark everything private by default. Then remove the private designation when that API is ready to be used by other code.

When to use an asserion?

We can use an assertion to protect our API from inproper callings by the syntaxassert(condition, output). It will print output and terminate the whole process when the condition is false

Extension

  1. The properties you add can have no storage associated with them (computed only).
  2. We use extension if and only if the extension makes this class a better class and the new property is concerned with that class.
  3. In extension, we can still use self key word to acces the instance that called the property. In the Int class or some other basic classes, self refer to the literal value self

Automatic Reference Counting

Mechanism

  1. Reference types (classes) are stored in the heap.
  2. How does the system know when to reclaim the memory for these from the heap?
    It “counts references” to each of them and when there are zero references, they get tossed. This is done automatically.
  3. It is known as “Automatic Reference Counting” and it is NOT garbage collection.

    Influencing ARC

  4. strong
    strong is “normal” reference counting
    As long as anyone, anywhere has a strong pointer to an instance, it will stay in the heap
  5. weak
    weak means “if no one else is interested in this, then neither am I, set me to nil in that case” Because it has to be nil-able, weak only applies to Optional pointers to reference types
    A weak pointer will NEVER keep an object in the heap
    Great example: outlets (strongly held by the view hierarchy, so outlets can be weak)
  6. unowned
    unowned means “don’t reference count this; crash if I’m wrong”
    This is very rarely used
    Usually only to break memory cycles between objects (more on that in a little while)

How to make optional methods in a Protocol?

Swift doesn’t provide this kind of methods, but we can accomplish it by make it an Objective-C Protocol. We need to add @objc before the protocol key word to mark that this is an Objective-C Protocol, and add optional key word in front of any methods we want to make optional.

NSAttributedString

NSAttributedString is another kind of String, which we can set its font, color… in code. To implement an NSAttributedString, we need to build a [NSAttributedStringKey : Any] dictionary first, the value of each key do not have a specific type, totally depend on the key. After that, we declare an NSAttributedString with its initializer NSAttributedString(String, [NSAttributedStringKey : Any]). Then we can assign this NSAttributedString to the attributedText or attributedLabel of a UILabel or a UIButton
