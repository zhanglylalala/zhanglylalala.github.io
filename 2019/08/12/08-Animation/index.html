08. Animation

Posted on 2019-08-12 Edited on 2020-08-07 In 计算机科学与技术 , Swift开发 (Stanford CS193p)

UIView Property Animation

UIView Animation Properties

  1. frame/center
  2. bounds (transient size, does not conflict with animating center)transform (translation,
  3. rotation and scale)
  4. alpha (opacity)
  5. backgroundColor

UIViewPropertyAnimator

  1. Use a static method of UIViewPropertyAnimation named runningPropertyAnimator and pass a bunch of parameters to it to set the animation we want.
  2. withDuration is the time the animation takes
  3. delay means that excute the animation so many seconds later
  4. options is the type of animation
  5. animations is a closure which takes nothing as parameter and returns nothing.
    This is where we will modify the properties.
    No matter when the animation will happen or end, this closure will be excuted immediately.
  6. completion is an optional parameter. It is also a closure which takes one UIViewAnimatingPosition as parameter and returns nothing
    It will be excuted when the animation is end.
    UIViewAnimatingPosition can be .start if we run the animation backwards, .end, and .current if this animation was interrupted by others
  7. If multiple animations involve in multiple differet properties, then they can be excuted at the same time. But is two animation involve in the same property, only the later animation will be executed and the easier one will stop immediately.
  8. The animation presented on screen will delay while codes will be excuted continuously.

UIViewAnimationOptions

  1. beginFromCurrentState: pick up from other, in-progress animations of these properties
  2. allowUserInteraction: allow gestures to get processed while animation is in progress
  3. layoutSubviews: animate the relayout of subviews with a parent’s animation
  4. repeat: repeat indefinitely
  5. autoreverse: play animation forwards, then backwards
  6. overrideInheritedDuration: if not set,use duration of any in-progress animation
  7. overrideInheritedCurve: if not set, use curve (e.g. ease-in/out) of in-progress animation
  8. allowAnimatedContent: if not set, just interpolate between current and end “bits”
  9. curveEaseInEaseOut: slower at the beginning, normal throughout, then slow at end
  10. curveEaseIn: slower at the beginning, but then constant through the rest
  11. curveLinear: same speed throughout

UIView Animation

  1. The difference between UIViewAnimation and UIViewPropertyAnimation is that UIViewAnimaiton can make an entire view modified at once.
  2. It sends message to the UIView instead of UIViewPropertyAnimation.
  3. In this case, we have a few more animation options to use.
    Flip the entire view over UIViewAnimationOptions.transitionFlipFrom{Left,Right,Top,Bottom}
    Dissolve from old to new state .transitionCrossDissolve
    Curling up or down .transitionCurl{Up,Down}

Dynamic Animation

  1. Set up physics relating animatable objects and let them run until they resolve to stasis.
  2. Easily possible to set it up so that stasis never occurs, but that could be performance problem.

Create a Dynamic Animation

  1. Create an animator : UIDynamicAnimator(referenceView: UIView)
    Every animating views must be in a view hierarchy with referenceView at the top.
  2. We need to give the animator some Behaviors, so we need to create some UIDynamicBehavior instances then use animator.addBehavior(…)
  3. To perform this animation, we also need to add items to the animator for it to perform. First we need to create some UIDynamicItem instances then add it to animator with syntax animator.addItem(item)
  4. UIView implements UIDynamicItem.
  5. If you change center or transform while the animator is running, you must call this method in updateItemUsingCurrentState(item: UIDynamicItem)

Behaviors

  1. All behaviors know the UIDynamicAnimator they are part of
  2. They can only be part of one at a time.
    var dynamicAnimator: UIDynamicAnimator? { get }
  3. The behavior will be sent this message when its animator changes …
    func willMove(to: UIDynamicAnimator?)

UIGravityBehavior

  1. var angle: CGFloat
    It’s in radians; 0 is to the right; positive numbers are clockwise
  2. var magnitude: CGFloat
    Every 1.0 is 1000 points/s2

UIAttachmentBehavior

  1. init(item: UIDynamicItem, attachedToAnchor: CGPoint)
  2. init(item: UIDynamicItem, attachedTo: UIDynamicItem)
  3. init(item: UIDynamicItem, offsetFromCenter: CGPoint, attachedTo[Anchor]...)
  4. var length: CGFloat
    Distance between attached things (this is settable while animating!)
  5. var anchorPoint: CGPoint
    Can also be set at any time, even while animating
  6. The attachment can oscillate like a spring and you can control frequency and damping

UICollisionBehavior

  1. var collisionMode: UICollisionBehaviorMode
    .items, then any items you add to a UICollisionBehavior will bounce off of each other
    .boundaries, then you add UIBezierPath boundaries for items to bounce off of
    It can also be .everything

  2. To add boundaries you need to use these methods
    func addBoundary(withIdentifier: NSCopying, for: UIBezierPath)
    func addBoundary(withIdentifier: NSCopying, from: CGPoint, to: CGPoint)
    func removeBoundary(withIdentifier: NSCopying)

    `var translatesReferenceBoundsIntoBoundary: Bool
    NSCopying means NSString or NSNumber, but remember you can as to String, Int.

UISnapBehavior

  1. It will ease the view to a specific place. And just like there are four springs around the item in the new spot, it will vibrate after get to the new spot.
  2. You can control the damping of these “four springs” with var damping: CGFloat
  3. init(item: UIDynamicItem, snapTo: CGPoint)

UIPushBehavior

  1. var mode: UIPushBehaviorMode
    .continuous or .instantaneous
  2. var pushDirection: CGVector
  3. var angle: CGFloat
    in radians and positive numbers are clockwise
  4. var magnitude: CGFloat
    magnitude 1.0 moves a 100x100 view at 100 pts/s2
  5. If you push .instantaneous, after it’s done it just sits there wasting memory.

UIDynamicItemBehavior

  1. Controls the behavior of items as they are affected by other behaviors.
  2. var allowsRotation: Bool
  3. var friction: CGFloat
  4. var elasticity: CGFloat
  5. Can also get information about items with this behaviors
    func linearVelocity(for: UIDynamicItem) -> CGPoint
    func addLinearVelocity(CGPoint, for: UIDynamicItem)
    func angularVelocity(for: UIDynamicItem) -> CGFloat

UIDynamicBehavior

  1. Superclass of behaviors.
  2. You can create your own subclass which is a combination of other behaviors.
  3. Usually you override init method(s) and addItem and removeItem to call.
  4. func addChildBehavior(UIDynamicBehavior)
    This is a good way to encapsulate a physics behavior that is a composite of other behaviors. We can use it in the initializer.
    You might also add some API which helps your subclass configure its children.
  5. All behaviors know the UIDynamicAnimator they are part of.
    They can only be part of one at a time.
    var dynamicAnimator: UIDynamicAnimator? { get }
    And the behavior will be sent this message when its animator changes
    func willMove(to: UIDynamicAnimator?)

UIDynamicBehavior’s action property

  1. Every time the behavior acts on items, this block of code that you can set is executed
    var action: (() -> Void)?
  2. You can set this to do anything you want. But it will be called a lot, so make it very efficient.
  3. If the action refers to properties in the behavior itself, watch out for memory cycles.

Stasis

  1. UIDynamicAnimator’s delegate tells you when animation pauses
  2. Just set the delegate
    var delegate: UIDynamicAnimatorDelegate
  3. You’ll find out when stasis is reached and when animation will resume
    func dynamicAnimatorDidPause(UIDynamicAnimator)
    func dynamicAnimatorWillResume(UIDynamicAnimator)

Memory Cycle Avoidance

Closure Capturing

  1. Closure can use the globle variables and can capture them.
  2. Closure can rename globle variables by adding [localName = globleName] in at the beginning.
  3. It can even change the influencing ARC of the local value, such as [weak localName = globleName]
  4. If we make localName and globleName to be the same, we only need to write the name like [weak globleName]

Memory Cycle

  1. Closures are reference types and live in the heap and they capture variables in their surroundings and keep them in the heap with them.
  2. Memory Cycle means that several strong values point to each other. So that none of them can leave the heap.
  3. A memory cycle is not good, because it will take too much memory.
  4. For exemple, a closure inside a class calls another method in that class. The closure captured the class, the class stored the closure. So the closure point to the class and class point back to the closure. It create a cycle. Neither can leave the heap.
  5. Remember that self is a strong variable.
  6. All the closures in animation we make will not contructe a memory cycle. Because those closures instead of point to the class they are in, they point to the animator.

Break a Memory Cycle

  1. We can break a memory cycle by capturing globle variables as weak or unowned.
  2. When declared weak, that local variable becomes an optional.
  3. If we do so, the local variables point to nothing and break the memory cycle.
  4. We use unowned if and only if we can sure this is the only time we will use it and we won’t use it after it’s set nil.
  5. We can capture self with syntax [weak self]
