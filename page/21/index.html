17. Core Motion and Camera

Posted on 2019-08-28 Edited on 2020-08-07 In 计算机科学与技术 , Swift开发 (Stanford CS193p)

Core Motion

Basic

  1. Core Motion is an API to access motion sensing hardware on your device.
  2. Its primary inputs are Accelerometer, Gyro, Magnetometer. Not all devices have all inputs.
  3. Class used to get this input is CMMotionManager. Use only one instance per application (else performance hit). It is a “global resource,” so getting one via a class method somewhere is okay
  4. There are two main usages of it. The first step of both usage is Check to see what hardware is available. Then we can either Start the sampling going and poll the motion manager for the latest sample it has or Set the rate at which you want data to be reported from the hardware and register a closure (and a queue to run it on) to call each time a sample is taken.

Alerts and Action Sheets

Difference

The two mechanisms are all used to pop up and ask the user something.

Alerts

  1. It pops up in the middle of the screen.
  2. Usually ask questions with only two answers (e.g. OK/Cancel, Yes/No, etc.).
  3. Can be disruptive to your user-interface, so use carefully.
  4. Often used for “asynchronous” problems (“connection reset” or “network fetch failed”).
  5. Can have a text field to get a quick answer (e.g. password)
  6. Example, sometimes when Settings needs to reconfirm your Apple ID.

Multithreading

Queue

  1. Multithreading is mostly about “queues”.
  2. Functions (usually closures, code to be run) are simply lined up in a queue.
  3. Then those functions are pulled off the queue and executed on an associated thread(s).

  4. There are two kinds of queues in Multithreading:
    Serial Queue: It has only one thread and excute one closure a time.
    Concurrent Queue: It has multiple threads and can execute several closures a time

React to touches

  1. We can react to touches with raw touch events. (touches, moves, up, etc.)
  2. But, we can also react to certain, predefined “gestures”. Gestures are recognized by instances of UIGestureRecognizer.
  3. There are two sides to using a gesture recognizer.
    First, add a gesture recognizer to a UIView
    Second, provide a method to “handle” that gesture
  4. Usually the first is done by a Controller. Though occasionally a UIView will do this itself if the gesture is integral to its existence.
  5. The second is provided either by the UIView or a Controller.
