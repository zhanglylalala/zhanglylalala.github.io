POJ 3614

Sunscreen

目标：尽量不要浪费防晒霜
定义不浪费：对当前奶牛可用的任意两瓶防晒霜$x$与$y$，如果出现其后的奶牛只出现$x$、$y$都可以用，$x$、$y$都不可以用，$x$可以用，$y$不可以用三种情况之一，则选择$y$
可选策略

  1. 按奶牛的$minSPF$升序排序，选择$SPF$最小的防晒霜，$SPF[y] < SPF[x]$
  2. 按奶牛的$minSPF$降序排序，选择$SPF$最大的防晒霜，$SPF[x]<SPF[y]$
  3. 按奶牛的$maxSPF$升序排序，选择$SPF$最小的防晒霜，$SPF[y]<SPF[x]$
  4. 按奶牛的$maxSPF$降序排序，选择$SPF$最大的防晒霜，$SPF[x]<SPF[y]$


可知，策略$2$是正确的

国王的游戏

每个人的金币$coins[i] = \displaystyle\frac{\prod^{i-1}_{x=0}minister[x].left}{minister[i].right}$
目标：$max(coins)$尽量小
只要把最优排列任意交换两个大臣都会使得答案变动，所有的交换都可以通过多次交换两个相邻大臣来实现，所以我们只需要考虑交换两个相邻大臣

  1. 交换前还是最优排列时，两个大臣的金币数分别时$\displaystyle\frac{\prod^{i-1}_{x=0}minister[x].left}{minister[i].right}$与$\displaystyle\frac{\prod^{i}_{x=0}minister[x].left}{minister[i+1].right}$
  2. 交换后他们的金币变为$\displaystyle\frac{\prod^{i-1}_{x=0}minister[x].left}{minister[i+1].right}$与$\displaystyle\frac{minister[i+1].left\times\prod^{i-1}_{x=0}minister[x].left}{minister[i].right}$
  3. 他们都有公因子$\displaystyle\prod^{i-1}_{x=0}minister[x].left$，所以最终是比较
    $max(\displaystyle\frac{1}{minister[i].right},\frac{minister[i].left}{minister[i+1].right})$与$max(\displaystyle\frac{1}{minister[i+1].right},\frac{minister[i+1].left}{minster[i].right})$
  4. 同乘$minister[i].right\times minister[i+1].right$即为比较
    $max(minister[i+1].right,minister[i].left\times minister[i].right)$与$max(minister[i].right, minister[i+1].left\times minister[i+1].right)$
  5. 又$minister[i+1].right≤minister[i+1].right\times minister[i+1].left$，$minister[i].right≤minister[i].right\times minister[i].left$
  6. 所以要使得交换前更优，则有$minister[i].left\times minister[i].right≤minister[i+1].left\times minister[i+1].right$
    所以贪心策略为按大臣左右手数字乘积升序排列

POJ 2054

由于有时间因子的存在，权值最大的点必须尽快上色，所以可以得出贪心的核心——权值最大的点必然会在其父节点被上色之后立即上色
对于树中的任意三个点$x$、$y$、$z$，假设$x$和$y$已经绑定，则只有三种染色顺序$x\rightarrow y\rightarrow z$和$z\rightarrow x\rightarrow y$
他们的代价分别是$x+2y+3z$和$z+2x+3y$
选择第一种染色顺序当且仅当$x+2y+3z≤z+2x+3y$，即$z≥\displaystyle\frac{x+y}2$
所以我们可以把$x$和$y$缩成一个点，并把他们权值的平均值作为新点的权值。每次都把单独权值最大的点和它的父节点缩成一个点，直到整个树都是一个点
这题有两种实现方法

  1. 在初始的时候就先叠加一个基础倍率，即把每个点的权值加一遍。
    后来每次缩点的时候，都加上子节点的权值与父节点中包含点的个数的乘积，意义是父节点染色后第几个染色，补上与父节点的倍率差
    再把权值叠加到父节点上，之后享受和父节点一样的倍率
  2. 还可以利用并查集和堆优化
    堆存储所有待缩的点，根节点没有父节点，不用缩到另一个点上，所以不用入堆
    利用并查集记录当前点缩到哪个点上

倍增

Posted on 2020-08-09 In 算法 , 基本算法

倍增

  1. 二分查找把查找的复杂度从$O(n)$降低到了，但是二分也有局限性，如果每次查找的结果都在数组的前几个位置的话，二分的效率还不如顺序查找的效率
  2. 倍增利用了二进制划分的性质。二进制划分就是“任意整数都可以表示成若干个$2$的次幂项的和”，我们只递推在$2$的整数次幂位置上的值。
  3. 二进制划分在快速幂中有巨大的作用

寻找区间

  1. 倍增的一个用处就是寻找左端点固定，满足一定条件的区间

  2. 设已确定在要求区间的子区间的右端点为$hi$，即我们已经确定$[lo,hi]$一定包含在答案区间内，要找到当前区间还要向右延伸的长度$len$

  3. 初始的时候$lo$就是固定的左端点，此时还未确定任何区间，所以$hi=lo$，从$2^0$开始尝试，$len = 2^0=1$

  4. 每次测试区间$[lo,hi+len]$是否合法，若合法，则可以确定这段区间是答案的子区间，$hi =hi+len$，$len=2*len$枚举下一个长度；若非法，则当前区间太长要缩减，$[lo,hi]$确定是合法的，不用动，把$len=len/2$即可

  5. 当$len=0$时，表示已找到答案区间，区间上界就是$hi$，退出循环

ST算法

  1. 处理问题：给定一个数组，在线回答区间$[l,r]$内的最大值

  2. 预处理：数组$F[i][j]$表示区间$[i,i+2^j-1]$内的最大值。递推的边界是$F[i][0]=arr[i]$。把每个区间拆成两个，可以得到$[i,i+2^{j-1}-1]$与$[i+2^{j-1},i+2^j-1]$，这两个是之前求过的区间，所以$F[i][j]=max(F[i][j-1],F[i+2^{j-1}][j-1])$

  3. 询问：求出满足$2^k<r-l+1≤2^{k+1}$的$k$。总长度是大于$2^k$的，所以$[l,l+2^k]$与$[r-2^k+1,r]$都不会超出区间。且总长度不大于$2\cdot2^k$，所以那两个长度为$2^k$的区间必然包括了整个区间。所以整个区间的最大值就是两个子区间的最大值中的最大值

前缀和与差分

Posted on 2020-08-07 Edited on 2020-08-21 In 算法 , 基本算法

前缀和

  1. 一维数组的前缀和：$sum[i]=a[i]+sum[i-1]$。一般下标从$1$开始，$0$作为哨兵，方便代码书写

  1. 二维矩阵中的二维前缀和：定义$sum[i][j]=\displaystyle\sum^i_{x=1}\sum^j_{y=1}a[x][y]$

  2. 易知$sum[i-1][j]+sum[i][j-1]$中$sum[i-1][j-1]$中的数据被计算了两次，所以要扣除一次，同时还少了一个点$a[i][j]$，所以有公式：$sum[i][j] = sum[i - 1][j] + sum[i][j-1]-sum[i-1][j-1]+a[i][j]$

  1. 推广上述公式可得，任意长$l$，宽$w$的矩形中的和为$sum[i][j]$扣除整个矩形上边冗余$sum[i-l][j]$和左边冗余$sum[i][j-w]$，同时补上被多减了一次的$sum[i-l][j-w]$，故$\displaystyle\sum^i_{x=i-l+1}\sum^j_{y=j-w+1}a[i][j]=sum[i][j]-sum[i-l][j]-sum[i][j-w]+sum[i-l][j-w]$

均分纸牌问题

  1. 有$M$个人排成一排，手中分别有$C[1]\sim C[M]$张纸牌，在每一步操作中，可以让某个人把自己手中的一张牌交给他旁边的一个人，问至少需要多少步操作才能让每个人手中的纸牌数相等
  2. 若一共有$T$张牌，则这个问题的终止状态就是每个人手中都有$\displaystyle\frac{T}{M}$张纸牌
  3. 第$1$个人要达到最终的状态只能把$C[1]-\displaystyle\frac{T}M$张牌给第$2$个人
  4. 第$2$个人此时有$C[2]+C[1]-\displaystyle\frac{T}M$张牌，此时第$1$个人已经饱和，不能再和第$1$个人交易，则只能把$C[2]+C[1]-2\times\displaystyle\frac{T}M$张牌给第$3$个人
  5. 依此类推，得到第$i$个人要把$\displaystyle\sum^i_{x=1}C[x]-i\times\displaystyle\frac{T}M$张牌给第$i+1$个人。特殊情况$i=M$时$\displaystyle\sum^M_{x=1}C[x]=M\times\frac{T}M=T$，无需特判
  6. 故需要的操作数为$\displaystyle\sum^M_{i=1}\Bigg|sum[i]-i\times\frac{T}M\Bigg|$
  7. 如果一开始我们就让所有人手里的牌都减掉$\displaystyle\frac{T}M$，即$A[i]=C[i]-\displaystyle\frac{T}M$。则最后要使得所有人手中的牌为$0$，则答案变为$\displaystyle\sum^M_{i=1}|sum[i]|$

差分

  1. 原数列$A$，差分数列$B$，则有$B[0] = A[0]$，$B[i] = A[i]-A[i - 1] (i>0)$
  2. 若对$A$数列中的$[l,r]$的所有元素都加$d$，则对$B$中有，$B[l]$加$d$，$B[r+1]$减$d$
  3. 差分操作是前缀和的逆运算，差分数列的前缀和是原数列，前缀和数列的差分是原数列

判断环

有一种办法就是利用拓扑排序，但这种办法只能静态判断，不能动态，下面还有一种办法：
在图中，新建一条边，判断这条边会不会导致形成一个环？
只需要开一个二维数组，记录两个点之间有没有连边(直接 / 间接)，新建了一条a -> b的边，就判断一下是不是已经有一条b -> a的边，有就会构成环。
边的更新：首先把graph[a][b] = 1，然后开始分点更新。先更新到a的边，遍历一遍，graph[i][a]不为0就说明有路径i -> a -> b，就可以更新graph[i][b] = 1。再更新b出发的边，遍历一遍如果有graph[b][i]不为0，就说明有a -> b -> i，就更新graph[a][i] = 1，但是还不能结束，还要遍历一遍到a的点，如果还有graph[j][a]不为0，就说明有j -> a -> b -> i，就更新graph[j][i] = 1

三分查找求单峰函数极值

  1. 要求单峰函数在极值的两侧都是严格单调，极大值左侧严格递增，右侧严格递减；极小值左侧严格递减，右侧严格递增。且只有一个极值

  2. 先考虑极大值情况，把区间用两个点$lmid$、$rmid$ 划分成三段区间

     如果$f(lmid)<f(rmid)$，则要么$lmid$在极大值左侧，$rmid$在极大值右侧，要么都在极大值的左侧，但极大值都在$lmid$的右侧，此时可以赋值$l=lmid$

     如果$f(rmid)<f(lmid)$，则要么$lmid$在极大值左侧，$rmid$在极大值右侧，要么都在极大值的右侧，但极大值都在$rmid$的左侧，此时可以赋值$r=rmid$

  3. 再考虑极小值情况

     如果$f(lmid)<f(rmid)$，则要么$lmid$在极大值左侧，$rmid$在极大值右侧，要么都在极大值的右侧，但极大值都在$rmid$的左侧，此时可以赋值$r=rmid$

     如果$f(rmid)<f(lmid)$，则要么$lmid$在极大值左侧，$rmid$在极大值右侧，要么都在极大值的右侧，但极大值都在$lmid$的右侧，此时可以赋值$l=lmid$

问题：给平面内一系列的点，找出其中距离最近的两个点
二分：先把所有点按$x$轴排序，分成左右两段，所有组合只有三种情况——两个点都在左半段；两个点都在右半段；一个点在左半段，一个点在右半段

STL

  1. lower_bound(begin, end, num)​：在数组区间$[begin, end)$中查找第一个大于或等于$num$的数，如果不存在，则返回$end$
  2. upper_bound(begin, end, num)​：在数组区间$[begin, end)$中查找第一个大于$num$的数，如果不存在，则返回$end$
  3. 这两个函数都是返回一个指针，减掉$begin$，就可以得到其与$begin$的相对位置，是一个整型
  4. 如果要找的是小于或等于查找值的最大位置的话
     使用lower_bound
      如果查找值存在，得到的位置就是等于查找值的位置
      如果查找值不存在，得到的是大于查找值的位置，且其上一个位置就是小于查找值的位置 (查找值不存在)，所以要减$1$
     使用upper_bound
      任何情况下得到的位置都是大于查找值，且其上一个位置等于查找值 (查找值存在)，或小于查找值 (查找值不存在)，所以都要减$1$
    综上，我们采用​upper_bound(begin, end, num) - begin - 1的策略
  5. 如果要找的是小于查找值的最大位置的话
     使用upper_bound
      如果查找值存在，得到的位置的上一个位置是查找值，再上一个位置才小于查找值，则需要减$2$
      如果查找值不存在，得到的是大于查找值的位置，且其上一个位置就是小于查找值的位置 (查找值不存在)，所以要减$1$
     使用lower_bound
      如果查找值存在，得到的位置是查找值，上一个位置小于查找值，要减$1$
      如果查找值不存在，得到的位置大于查找值，但由于查找值不存在，其上一个位置小于查找值，要减$1$
    综上，我们采用lower_bound(begin, end, num) - begin - 1的策略

离散化

  1. 把所有数据排序去重之后，从第一个开始逐一往后标记，把每一个数据和一个整数一一对应
  2. 排序之后再离散化可以保留原始数据的大小关系，不必查询原数据，就可以直接从Hash后的值得到其大小关系

例如POJ 3179

  1. 首先很明显，这题要用二维前缀和来求解一个区域内的和

  2. 答案直接求不容易，但是判定一个值是不是答案很容易，所以可以二分答案求解

  3. 这题对空间限制比较严格，而且只有少部分的地方有非$0$值，所以可以把所有非$0$值的横纵坐标离散化，$500$个点离散后最大只有$1000$，来压缩空间

  4. 离散化后，在确认答案时，要在哈希表中找到原来的坐标对应的哈希后的坐标以及遍历的上界
    找哈希后的坐标要求的是不大于目标坐标的点，只要找出所有在当前区域内的和即可
    遍历上界的要求是不超过哈希表中最大元素

中位数

  1. 在有序序列中，若下标是从$1$开始的，则当总共有奇数个数时，中位数的下标为$\displaystyle\frac{n+1}{2}$，总共有偶数个数时，中位数是下标为$\displaystyle\frac{n}{2}+1$与$\displaystyle\frac{n}{2}$的数之间的数；若下标是从$0$开始的，则当总共有奇数个数时，中位数的下标为$\displaystyle\frac{n-1}{2}$，当总共有偶数个数时，中位数是下标为$\displaystyle\frac{n}2-1$与$\displaystyle\frac{n}{2}$
  2. 在一个数组中考虑$\displaystyle\sum |a[i]-x|$的最小值。假设在数轴中$a$数组有$P$个数比$x$小，有$Q$个数比$x$大
    若$P若$P>Q$，则每把$x-1$，要求的值都会减小$P-Q$
    所以使得$\displaystyle\sum|a[i]-x|$最小的$x$就是$a$数组的中位数，若$a$数组有偶数个元素，则$x$为两个中位数之间的任意数，若有奇数个元素，则$x$为中位数本身

环形均分纸牌问题

  1. 环形均分纸牌问题只是在均分纸牌问题上取消一个限制，最后一个人可以把纸牌给第$1$个人
  2. 从均分纸牌问题的推导中，我们可以知道必然有一个人不用给任何人纸牌，这个人就可以被看作是均分纸牌问题中的最后一个人，即环形链从这个人断开。在环形均分纸牌问题中，我们只需要枚举每一个人作为最后一个人的情况，取其结果的最小值就是答案
  3. 先给每个人从$1$号标到$M$号，记$A[1\sim M]=C[i\sim M]-\displaystyle\frac{T}M$，$A$数组的前缀和$sum[1\sim M]$，则当第$k$个人为最后一个人，链从这里断开时，前缀和
    $sum_k[k+1]=sum[k+1]-sum[k]$
    $sum_k[k+2]=sum[k+2]-sum[k]$
    $…$
    $sum_k[M]=sum[M]-sum[k]$
    $sum_k[1]=sum[1]+sum_k[M]=sum[1]+sum[M]-sum[k]$
    $sum_k[M]=sum[k]+sum[M]-sum[k]=sum[M]$
  4. 故答案为$\displaystyle\sum^M_{i=1}\bigg|sum[i]-sum[k]\bigg|$的最小值，即为$sum$数组的中位数

动态维护中位数

  1. 在线的做法是对顶堆算法

  2. 把序列中从小到大的$1\sim\displaystyle\frac{M}2+1$都存储在大根堆中，$\displaystyle\frac{M}2+2\sim M$存储在小根堆中
    小根堆中共有$\displaystyle\frac{M}2+1$个数，大根堆中一共有$\displaystyle\frac{M}2-1$个元素

  3. 每读入一个数，如果比大根堆的堆顶元素小，则推入大根堆，否则推入小根堆。如果有一个堆的大小不符合要求的，就把太大的那个堆的堆顶元素弹出，推入另一个堆中

  4. 中位数就是小根堆的堆顶元素

逆序对

  1. 在冒泡排序中，每一次交换就是修复一对逆序对，所以冒泡排序中的交换次数就是逆序对数

  2. 但是冒泡排序的复杂度是$O(n^2)$，相对较高，可以利用归并排序优化

  3. 在归并排序的归并过程中$loPtr$指向当前正在排序数组段的前半段中正在比较的数，$hiPtr$指向后半段正在比较的数。若$hiPtr$指向的数小于$loPtr$指向的数，则$hiPtr$指向的数与从$loPtr$到$mid$之间的所有数都构成逆序对

数码问题

  1. 在$n\times m$的矩阵中，有$n\times m-1$个数和一个空格，空格可以和其上下左右相邻的格子交换，给出两个局面，问能否从其中一个局面变换到另一个局面
  2. 将矩阵内的非空格内容展开成一维数组，空格的左右交换相当于没有操作，不会改变一维数组，但空格向上交换相当于一维数组中被交换的那格向后移动了$m-1$位；向下交换相当于前移了$m-1$位
  3. 若$m$为奇数，则$m-1$为偶数，移动的过程消除了$x$个逆序对，但增加了$y$个逆序对 (为方便相等的也记为新的逆序对)，且$x+y=m-1$。逆序对的数量变化为$y-x$。$x+y$是偶数，则$y-x$也是偶数。$x$与$y$要么都是奇数，要么都是偶数
    所以若两个局面可以相互转换，则他们的一维展开逆序对的奇偶性一样
  4. 若$m$为偶数，则$m-1$为奇数。则$y-x$也为奇数，则每变动一行奇偶性变动一次，即逆序对数量之差的奇偶性与行数之差的奇偶性相同
