在图中，新建一条边，判断这条边会不会导致形成一个环？
只需要开一个二维数组，记录两个点之间有没有连边(直接 / 间接)，新建了一条a -> b的边，就判断一下是不是已经有一条b -> a的边，有就会构成环。
边的更新：首先把graph[a][b] = 1，然后开始分点更新。先更新到a的边，遍历一遍，graph[i][a]不为0就说明有路径i -> a -> b，就可以更新graph[i][b] = 1。再更新b出发的边，遍历一遍如果有graph[b][i]不为0，就说明有a -> b -> i，就更新graph[a][i] = 1，但是还不能结束，还要遍历一遍到a的点，如果还有graph[j][a]不为0，就说明有j -> a -> b -> i，就更新graph[j][i] = 1

三分查找求单峰函数极值

  1. 要求单峰函数在极值的两侧都是严格单调，极大值左侧严格递增，右侧严格递减；极小值左侧严格递减，右侧严格递增。且只有一个极值

  2. 先考虑极大值情况，把区间用两个点$lmid$、$rmid$ 划分成三段区间

     如果$f(lmid)<f(rmid)$，则要么$lmid$在极大值左侧，$rmid$在极大值右侧，要么都在极大值的左侧，但极大值都在$lmid$的右侧，此时可以赋值$l=lmid$

     如果$f(rmid)<f(lmid)$，则要么$lmid$在极大值左侧，$rmid$在极大值右侧，要么都在极大值的右侧，但极大值都在$rmid$的左侧，此时可以赋值$r=rmid$

  3. 再考虑极小值情况

     如果$f(lmid)<f(rmid)$，则要么$lmid$在极大值左侧，$rmid$在极大值右侧，要么都在极大值的右侧，但极大值都在$rmid$的左侧，此时可以赋值$r=rmid$

     如果$f(rmid)<f(lmid)$，则要么$lmid$在极大值左侧，$rmid$在极大值右侧，要么都在极大值的右侧，但极大值都在$lmid$的右侧，此时可以赋值$l=lmid$

STL

  1. lower_bound(begin, end, num)​：在数组区间$[begin, end)$中查找第一个大于或等于$num$的数，如果不存在，则返回$end$
  2. upper_bound(begin, end, num)​：在数组区间$[begin, end)$中查找第一个大于$num$的数，如果不存在，则返回$end$
  3. 这两个函数都是返回一个指针，减掉$begin$，就可以得到其与$begin$的相对位置，是一个整型
  4. 如果要找的是小于或等于查找值的最大位置的话
     使用lower_bound
      如果查找值存在，得到的位置就是等于查找值的位置
      如果查找值不存在，得到的是大于查找值的位置，且其上一个位置就是小于查找值的位置 (查找值不存在)，所以要减$1$
     使用upper_bound
      任何情况下得到的位置都是大于查找值，且其上一个位置等于查找值 (查找值存在)，或小于查找值 (查找值不存在)，所以都要减$1$
    综上，我们采用​upper_bound(begin, end, num) - begin - 1的策略
  5. 如果要找的是小于查找值的最大位置的话
     使用upper_bound
      如果查找值存在，得到的位置的上一个位置是查找值，再上一个位置才小于查找值，则需要减$2$
      如果查找值不存在，得到的是大于查找值的位置，且其上一个位置就是小于查找值的位置 (查找值不存在)，所以要减$1$
     使用lower_bound
      如果查找值存在，得到的位置是查找值，上一个位置小于查找值，要减$1$
      如果查找值不存在，得到的位置大于查找值，但由于查找值不存在，其上一个位置小于查找值，要减$1$
    综上，我们采用lower_bound(begin, end, num) - begin - 1的策略

离散化

  1. 把所有数据排序去重之后，从第一个开始逐一往后标记，把每一个数据和一个整数一一对应
  2. 排序之后再离散化可以保留原始数据的大小关系，不必查询原数据，就可以直接从Hash后的值得到其大小关系

例如POJ 3179

  1. 首先很明显，这题要用二维前缀和来求解一个区域内的和

  2. 答案直接求不容易，但是判定一个值是不是答案很容易，所以可以二分答案求解

  3. 这题对空间限制比较严格，而且只有少部分的地方有非$0$值，所以可以把所有非$0$值的横纵坐标离散化，$500$个点离散后最大只有$1000$，来压缩空间

  4. 离散化后，在确认答案时，要在哈希表中找到原来的坐标对应的哈希后的坐标以及遍历的上界
    找哈希后的坐标要求的是不大于目标坐标的点，只要找出所有在当前区域内的和即可
    遍历上界的要求是不超过哈希表中最大元素

中位数

  1. 在有序序列中，若下标是从$1$开始的，则当总共有奇数个数时，中位数的下标为$\displaystyle\frac{n+1}{2}$，总共有偶数个数时，中位数是下标为$\displaystyle\frac{n}{2}+1$与$\displaystyle\frac{n}{2}$的数之间的数；若下标是从$0$开始的，则当总共有奇数个数时，中位数的下标为$\displaystyle\frac{n-1}{2}$，当总共有偶数个数时，中位数是下标为$\displaystyle\frac{n}2-1$与$\displaystyle\frac{n}{2}$
  2. 在一个数组中考虑$\displaystyle\sum |a[i]-x|$的最小值。假设在数轴中$a$数组有$P$个数比$x$小，有$Q$个数比$x$大
    若$P若$P>Q$，则每把$x-1$，要求的值都会减小$P-Q$
    所以使得$\displaystyle\sum|a[i]-x|$最小的$x$就是$a$数组的中位数，若$a$数组有偶数个元素，则$x$为两个中位数之间的任意数，若有奇数个元素，则$x$为中位数本身

环形均分纸牌问题

  1. 环形均分纸牌问题只是在均分纸牌问题上取消一个限制，最后一个人可以把纸牌给第$1$个人
  2. 从均分纸牌问题的推导中，我们可以知道必然有一个人不用给任何人纸牌，这个人就可以被看作是均分纸牌问题中的最后一个人，即环形链从这个人断开。在环形均分纸牌问题中，我们只需要枚举每一个人作为最后一个人的情况，取其结果的最小值就是答案
  3. 先给每个人从$1$号标到$M$号，记$A[1\sim M]=C[i\sim M]-\displaystyle\frac{T}M$，$A$数组的前缀和$sum[1\sim M]$，则当第$k$个人为最后一个人，链从这里断开时，前缀和
    $sum_k[k+1]=sum[k+1]-sum[k]$
    $sum_k[k+2]=sum[k+2]-sum[k]$
    $…$
    $sum_k[M]=sum[M]-sum[k]$
    $sum_k[1]=sum[1]+sum_k[M]=sum[1]+sum[M]-sum[k]$
    $sum_k[M]=sum[k]+sum[M]-sum[k]=sum[M]$
  4. 故答案为$\displaystyle\sum^M_{i=1}\bigg|sum[i]-sum[k]\bigg|$的最小值，即为$sum$数组的中位数

动态维护中位数

  1. 在线的做法是对顶堆算法

  2. 把序列中从小到大的$1\sim\displaystyle\frac{M}2+1$都存储在大根堆中，$\displaystyle\frac{M}2+2\sim M$存储在小根堆中
    小根堆中共有$\displaystyle\frac{M}2+1$个数，大根堆中一共有$\displaystyle\frac{M}2-1$个元素

  3. 每读入一个数，如果比大根堆的堆顶元素小，则推入大根堆，否则推入小根堆。如果有一个堆的大小不符合要求的，就把太大的那个堆的堆顶元素弹出，推入另一个堆中

  4. 中位数就是小根堆的堆顶元素

逆序对

  1. 在冒泡排序中，每一次交换就是修复一对逆序对，所以冒泡排序中的交换次数就是逆序对数

  2. 但是冒泡排序的复杂度是$O(n^2)$，相对较高，可以利用归并排序优化

  3. 在归并排序的归并过程中$loPtr$指向当前正在排序数组段的前半段中正在比较的数，$hiPtr$指向后半段正在比较的数。若$hiPtr$指向的数小于$loPtr$指向的数，则$hiPtr$指向的数与从$loPtr$到$mid$之间的所有数都构成逆序对

数码问题

  1. 在$n\times m$的矩阵中，有$n\times m-1$个数和一个空格，空格可以和其上下左右相邻的格子交换，给出两个局面，问能否从其中一个局面变换到另一个局面
  2. 将矩阵内的非空格内容展开成一维数组，空格的左右交换相当于没有操作，不会改变一维数组，但空格向上交换相当于一维数组中被交换的那格向后移动了$m-1$位；向下交换相当于前移了$m-1$位
  3. 若$m$为奇数，则$m-1$为偶数，移动的过程消除了$x$个逆序对，但增加了$y$个逆序对 (为方便相等的也记为新的逆序对)，且$x+y=m-1$。逆序对的数量变化为$y-x$。$x+y$是偶数，则$y-x$也是偶数。$x$与$y$要么都是奇数，要么都是偶数
    所以若两个局面可以相互转换，则他们的一维展开逆序对的奇偶性一样
  4. 若$m$为偶数，则$m-1$为奇数。则$y-x$也为奇数，则每变动一行奇偶性变动一次，即逆序对数量之差的奇偶性与行数之差的奇偶性相同

最短Hamilton路径

  1. $Hamilton$路径的定义是从$0$到$n-1$不重不漏地经过每一个点恰好一次

  2. 朴素的算法是枚举$n$个点遍历顺序的全排列，复杂度为$O(n\cdot n!)$

  3. 利用状态压缩可以降低复杂度到$O(n^2\cdot 2^n)$

    定义整数$i$，在$i$的二进制表示中从右到左依次为第$0$位到第$n-1$位，若第$k$位为$1$，则表示第$k$个点已经遍历过；若第$k$位为$0$，则表示第$k$个点仍未遍历

    定义状态$dp[i][j]$表示当前的状态整数为$i$且当前正在第$j$个点，则有要求$i$的第$j$位为$1$

    如果记数字$i’$为另一个状态，其与$i$的差别仅仅是$i$的第$j$位为$1$，$i’$的第$j$位为$0$，那状态$dp[i][j]$就可以由$dp[i’][k]+map[k][j]$得到，其中$i’$的第$k$位为$1$

    所以有状态转移方程

图表示法

抽象语法树

  1. 抽象语法：语言结构本质性的东西，不考虑具体符号表示
  2. 抽象语法树：反映抽象语法，是分析书的抽象形式或压缩形式
  3. 抽象语法树中每一个叶节点都表示运算对象，内部节点表示运算符
  4. 分析树反映的是具体语法结构
  5. 语法规则中符号可能起标点符号作用，也可能起解释作用
  6. 辅助函数：
     建立一个运算符节点
     建立一个标识符节点，即变量下存储的运算对象，变量名为$id$，变量的值为$value$
     建立一个数节点，标号为$num$，数的值为$val$
  7. 利用语法制导定义，每一个产生式的语义规则都是其对应节点的产生辅助函数。如果产生式是$E\to id$，就用；若是$E\to num$，就用；其它的就用来产生对应的运算符
  8. 利用图像直观地展示从分析树生成语法树的过程就可以用箭头从一个语法树的节点指向生成的那个节点，并将节点中的域指针指向对应的语法树节点。最后再把分析树和从分析树指向语法树的箭头去掉，就得到一个语法树

有向非循环图

  1. 在抽象语法树中，有大量节点表示的是同一个值，这造成了大量的空间浪费
  2. 我们可以把抽象语法树中的公共节点提取出来，只建立一个节点，并把原本指向不同的同值节点的域指针都指向那一个公共节点
  3. 显然这样产生的图就不再是一棵树，一个公共节点可能有多个父节点
  4. 我们相当于只是把原本语法树的节点合并成一个，并没有改变其走向，所以这个图不会有循环
  5. 现在在构造节点前要先检查现有节点，若存在相同节点，就直接返回，确定了不存在相同节点才会创建新的节点

逆波兰表达式

  1. $\Theta$是$k$目运算符，则$\Theta$作用于$e_1e_2…e_k$的结果用$e_1e_2…e_k\Theta$表示
  2. 对语法树进行一次后根遍历得到的序列就是后缀式
  3. 还是利用语法制导定义，设置每一个产生式的语义规则为给左部赋予一个$code$属性，表示改产生式的后缀式。用$||$表示空格，将符号串连接起来
  4. 后缀式的优点是易于计算机处理表达式
  5. 常见的是栈算法，详见 栈和队列

类型检查

  1. 类型检查是语义分析的重要组成部分
  2. 编译器先根据类型说明，确定每一个变量和常量的类型，计算其使用存储空间大小，从而建立其到存储空间的映射。进而，编译器要确定每个语言结构的类型，判定重载算符 (函数)在程序中代表哪个、类型转换、类型检查
  3. 类型表达式：
     类型名和基本类型是类型表达式
     类型构造符作用于类型表达式的结果仍然是类型表达式
      数组构造符$ARRAY(I,T)$，$I$是下标集合，$T$是数组类型
      笛卡尔乘积$\times$，
      记录类型构造符$RECORD$，其结果类似$C$中的结构体
      指针类型构造符$POINTER$，其结果是一个指针类型
      函数类型构造符$\to$，$D_1\times D_2\times…\times D_n\to R$
     类型表达式中可以出现类型变量，类型变量的值科院是任何类型表达式
  4. 分类
     静态类型检查：由编译器完成的检查
     动态类型检查：目标程序运行时完成的检查。若目标代码把每个对象的类型和值一起保存，那么任何检查都可以动态完成
  5. 强类型语言：保证编译器所接受的程序不会有运行时的类型错误的语言
  6. 类型检查的内容：表达式、语句、函数
  7. 变量标识符类型的确定
     语义分析程序先处理类型说明，建立类型表达式，然后处理变量说明，建立变量和类型表达式的绑定
     把变量标识符的类型信息记录在其符号表的表项中，$addtype(id.entry,T.type)$完成这个任务，引入综合属性$T.type$记录类型表达式
  8. 表达式的类型检查：检查运算对象之间是否满足相容条件。综合属性$E.type$表示$E$的类型表达式。常用$IF…THEN…ELSE$或$lookup(id.entry)$
  9. 语句的类型检查：主要包括赋值语句类型的相容性、控制表达式的结果类型检查。语句中没有类型错误就是$void$，否则为$type_error$。语句中有表达式，所以还要对表达式类型检查
  10. 函数引用的类型检查：检查参数是否和函数类型一致，是则返回函数返回值的类型
  11. 必须类型转换：赋值号右边的类型转换为左边的类型；整数转换成实数
  12. 类型表达式在源程序的中间表示里插入这些转换操作
  13. 自动类型转换 (隐式转换)：从一种数据类型转换成另一种数据类型，可以由编译器自动完成
    强制类型转换 (显式转换)：必须由程序员显式地写在源程序中，在类型检查器中类似函数调用

常用语句的翻译

说明语句的翻译

  1. 说明语句：对程序中规定范围内使用的各类变量、常数、过程进行说明
  2. 编译要做的工作：在符号表中建立相应的表项，填写有关的信息。如类型、嵌套深度、相对地址
  3. $T.width$记录名字的域宽，$T.type$记录名字的类型，$enter(id.name,T.type,offset)$为名字$name$建立一个符号表表项
  4. 用一个全局变量$offset$来记录下一个可用单元相对于初始单元的相对地址。每新建一个变量，就给$offset$叠加该变量的域宽

保留作用域信息

  1. 同一个标识符有可能在不同的范围内表示不同的信息，要正确区分在各个范围内引用的标识符而不产生混乱
  2. 首先要解决的问题是局部数据和非局部数据的访问
  3. 方法是为每个过程建立一张符号表，每张表都有一个指针指向其直接外围过程的符号表，如果内部数据中无法找到，就顺着指针去直接外围过程符号表找
  4. 子过程名$id$作为直接外围过程的局部名字记录在直接外围过程符号表中，其中还有一个指针指向子过程$id$
  5. 每次生成一个过程 (包括主程序)，先执行定位操作，记录所需空间，若不是主程序，还要返回到外围过程，再执行重定位操作，创建主出现的符号表、初始化
  6. 翻译过程用到一个栈存放指向外围过程的指针。还有一个栈存放变量的相对地址，当过程结束时，里面记录的就是该过程占用的所有字节数
  7. 辅助操作
     $mktable(previous)$创建一张新符号表
     $enter(table,name,type,offset)$插入表项
     $addwidth(table,width)$记录总域宽
     $enterproc(table,name,newtable)$进入过程
  8. 记录声明：定位操作中建立一个空表记录各个域中的信息，最后的重定位操作中将各域的总域宽作为该记录型的域宽，返回记录声明所在过程的符号表

赋值语句的翻译

  1. 利用过程$lookup(id.name)$检查是否在符号表中存在相应次名字的表项
  2. $lookup$函数先通过$top(tblptr)$指针在当前符号表中查找，若找到直接返回；若没找到，就到当前表的外围表找，直到找到返回，或所有外围表项都没找到，返回$nil$
  3. 查找只要对终结符中的标识符查找，不必对非终结符或终结符中的常量查找
  4. 语义动作应包括类型分析，文法符阿訇应有类型属性，在类型分析的基础上，进行相容和赋值相容检查，生成类型转换的三地址代码
  5. 一维数组$A$的基址为$base$，域宽为$w$，下标下界为$low$，上界为$high$。$A[i]$的位置为$i\times w+base-low\times w$
    二维数组行列下界分别为$low_1,low_2$，长度$n_1,n_2$，$A[i][j]$位置为$(i\times n_2 + j)\times w + base-(low_1\times n_2+low_2)\times w$
    $A[i_1,i_2,…,i_k]$的地址为$((…((i_1\times n_2+i_2)\times n_3+i_3)…)\times n_k+i_k)\times w+base-((…((low_1\times n_2+low_2)\times n_3+low_3)…)\times n_k+low_k)\times w$
  6. 我们用$Elist$来表示数组的一个下标，多个下标用逗号分割。定义其属性$Elist.array$为数组名在符号表的位置，$Elist.ndim$为目前已经识别出的下标表达式的个数，$Elist.place$保存递推公式中临时变量位置
  7. 如果一个变量的$.offset$属性是$null$，则为简单变量，如果是一个变项，则是一个数字元素，且其值为$Elist.array$
  8. 访问记录中的域：先找到符号表里的记录变量的位置，再找到记录表的位置

布尔表达式和控制流语句

  1. 布尔表达式与一般的算术表达式类似，运算符是与或非等，运算量是布尔常量或关系表达式，运算结果是布尔常量
  2. 布尔表达式的值的表示方法：数值法、控制流表示法。布尔表达式的翻译方法也是这两种
  3. 数值翻译法就只有一个属性$E.place$存放$E$的布尔数值，控制流翻译法就有$E.true$表示$E$值为真时应执行的第一条语句的标号，$E.false$表示$E$值为假时应执行的第一条语句的标号。他们都有$E.code$表示三地址代码
  4. 对语句$S$有：$S.next$表示紧跟在语句$S$之后的三地址语句的标号，$S.begin$表示语句$S$的第一条三地址语句的标号，$S.code$表示语句$S$的三地址代码

回填技术

  1. 两遍扫描的翻译技术 ($L$属性定义，深度优先翻译)：先产生分析树，再为分析树加注释 (翻译)
  2. 一边扫描技术 ($S$属性定义，自底向上翻译)的问题时生成某些转移指令时，目标地址可能还不知道，即布尔表达式的真出口和假出口可能还不知道
  3. 解决：先产生无转移标号的转移语句，再转向标号确定后，再把语句标号填上，即回填
  4. 定义综合属性$E.truelist$记录转移到$E$的真出口的指令的链表指针，$E.falselist$记录转移到$E$的假出口的指令的链表指针。一个真出口链表里可能有不止一个指针，但他们指向的语句的跳转出口都是一样的
  5. 在计算时，我们先把所有带回填且需回填同一个值的指针串成一条链，当满足回填要求的时候就把其中的每一个都填上
